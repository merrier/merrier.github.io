## 虚拟滚动速度太快会短暂白屏 问题解决

加快渲染速度
    CSS 设置优化： will-change，CSS Container
    缓存 HTMLElement，预先加载好部分HTMLElement（代价是内存开销）
给虚拟滚动增加一定 Buffer
滚动发生后如果视口超出 Buffer 就同步地进行渲染（不 Debouce / Throttle）

### 图片循环滑动

在最后一张图片切换到第一张图片的时候，期待平滑无感知；由于本需求比较简单，从左向右的图片切换，且无用户交互。所以基本解决方法就是，**在最后多加一张图片A，图片D运动到最后一个A以后，视窗从最后一个位置A迅速切换到第一个位置A**，此过程用户无感知，然后开启下一次的循环。
